4/19

  • Most commonly optimization problems (sign of DP)

  • When re-ordering doesn’t change the final result but simplifies the solution
    • Finding the best subset that satisfies some constraints
    • Scheduling problems
    • Variants of the longest increasing sequence problem

2. Examples

6/19

  • Russian Doll Envelopes
  • Observation 1: Envelope 1 can fit inside envelope 2 when: w1 < w2 and h1 < h2.
    • A sequence of envelopes form an increasing sequence in both width and length.
  • Observation 2: Sorting by width/length reduces the problem to finding LIS in 1D.
  • Example: [[2, 2], [6, 5], [6, 4], [5, 3]]
  • [2, 2], [5, 3], [6, 5], [6, 4]
  • [2, 3, 5, 4]

7/19

  • Maximum Profit in Job Scheduling
  • Observation 1: Similar to the previous problem, a set of valid jobs form an increasing start/end time.
  • Observation 2: When consider a job (s, e), we must look at all jobs ending no later than s => sorting by endtime simplifies the problem.

=> O(n^2) LIS-like solution.

  • Observation 3: if job j comes after job i and f[j] <= f[i], we can forget about j
    • The DP table forms an increasing sequence that we can binary search on.

=> O(n log n) solution.

  • Minimum Total Distance Traveled
  • Observation 1: a factory with limit k = k factories with limit 1.
  • Observation 2: if robot 1 is to the left of robot 2 and factory 1 is to the left of factory 2, it never makes sense to assign robot1 to factory 2 and robot 2 to factory 1.

=> Sort robots and factories and we have a classic DP problem.

16/19

  • Make Array Strictly Increasing
  • Observation 1: When replacing a number, we always look for something just a bit bigger than the previous one.
    • Example: arr1: [1, 5, 3, 6, 7], arr2: [1, 3, 2, 4]

=> Sort arr2.

  • Observation 2: Try 1D DP: f[i] = min no. operations to make arr1[0…i] increasing.

=> Impossible without knowing the previous number, must expand DP state.

  • Observation 3: Each number in arr1 can only be unchanged or one of the values in arr2 => max len(arr2) + 1 values.

=> DP state: f[i][j] = min no. to make arr1[0…i] increasing with the last number being arr2[j] (with j = len(arr2) meaning the last number is unchanged).

[EngineerPro] Sorting & DP

17/19

  • Let m = len(arr2)
  • f[i][j] = min of
    • f[i - 1][m] + 1 if arr1[i - 1] < arr2[j]
    • f[i - 1][0…j-1] + 1 (assuming arr2 have distinct values)
  • f[i][m] = min of
    • f[i - 1][m] if arr1[i -1] < arr1[i]
    • f[i - 1][j] (j = 0…m-1) if arr2[j] < arr1[i]
  • Observation 4: f[i][0], f[i][1], …, f[i][m - 1] is an non-increasing sequence
    • f[i][j] = min of
      • f[i - 1][m] + 1 if arr1[i - 1] < arr2[j]
      • f[i - 1][j-1] + 1 (assuming arr2 have distinct values)
