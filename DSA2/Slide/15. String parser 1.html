2.1.1 Basic calculator II

  • Observation

+ We do not need to deal with parenthesis

+ We only need to deal with +,-,*,/

+ We must prioritize *, / operators first, and then will come to +, - operators later

(E.g: 5 + 10*6 = 5 + 60 = 65)

=> Parse string (from left to right) and store numbers in a stack. If we encounter *, /, we will do this calculation immediately with the top value of stack. If we encounter +, - => just push number in the stack only.

After finish all steps above, the final value is the summation of all numbers from stack

3. Homework

  • (Google Question)

We define 2 operator:

- a b # => (a + b) * b (e.g: 3 5 # => (3 + 5) * 5 = 40

- a b ? => (a - b) * b (e.g: 3 5 ? => (3 - 5) * 5 = -10

- a b @ => (a - 3 * b) * a (e.g: 3 5 @ => (3 - 3 * 5) * 3 = -36

Give you a string of a valid expression => let calculate the final value

We do this calculation from left to right, prioritize the expression inside parentheses first. For operators, they are the same level of priority

(expression contains only integer, space, (, ), and operators: #, ?, @)

(length of input <= 10^ 6)

For example: input: (1 1 #) (3 (2 2 ?) #) @

Output: => 2 (3 0 #) @ => 2 0 @ => 4
